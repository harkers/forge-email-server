# DevForge: Project Foundry
## Approval Gates

Status: Planning  
Date: 2026-03-28

## Gate Model

Project Foundry should not move from planning into implementation simply because the concept feels strong. It should pass explicit approval gates.

## Gates

### G-0 — Concept
Purpose: confirm the problem is real and the intended outcome is worth pursuing.

Expected evidence:
- problem statement
- success metrics
- initial stakeholder alignment

### G-1 — Charter
Purpose: confirm the charter, scope, constraints, and initial structure are accepted.

Expected evidence:
- approved charter
- agreed scope
- initial RACI
- current phased plan

### G-2 — Funding
Purpose: confirm the work can be resourced and supported.

Expected evidence:
- budget rough order of magnitude
- sponsor agreement
- delivery expectation confirmed

### G-3 — Governance Readiness
Purpose: confirm that the minimum human governance layer is defined and that the AI operating model has a clear approval structure.

Expected evidence:
- executive sponsor / human authority defined
- product decision owner defined
- governance approval owner defined
- privacy approval path defined
- architecture oversight path defined
- AI operating role model documented clearly enough to proceed

### G-4 — Repo Initiation
Purpose: confirm the planning pack is good enough to begin repository bootstrap.

Expected evidence:
- planning pack reviewed
- gate owners satisfied
- no blocking unresolved concept issues
- approval to begin Sprint 0

## Rule

No template repository bootstrap before G-4 is complete.
