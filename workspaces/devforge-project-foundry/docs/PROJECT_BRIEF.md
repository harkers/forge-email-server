# DevForge: Project Foundry
## Full Project Brief

Version: 0.9  
Date: 2026-03-28  
Status: Planning

## 1. Overview

DevForge: Project Foundry is the master OpenClaw workspace and future GitHub template repository used to create governed project environments in a consistent, repeatable way.

Every project created from this foundation should inherit structured lifecycle stages, forge-prefixed execution domains, built-in governance thinking, dedicated communication and training capability, and a living risk cycle.

It is not just a repository scaffold. It is a governed delivery operating foundation.

## 2. Problem Statement

New projects often begin as empty repositories, ad-hoc notes, or loosely defined task boards. That leads to fragmented documentation, weak traceability, inconsistent governance, late security and privacy review, poor handoffs, weak stakeholder communication, and training material being produced too late.

Project Foundry exists to remove that waste and replace it with a reusable project foundation that is structured from strategy through implementation, review, release, enablement, and closure.

## 3. Vision

Create a master OpenClaw project foundation that allows teams to spin up world-class, governed project environments quickly without sacrificing quality, control, traceability, or clarity.

## 4. Goals

- Create a repeatable project foundation for OpenClaw workspaces
- Standardise project structure across strategy, architecture, governance, delivery, review, release, communications, training, and risk
- Ensure every future project starts with built-in templates, directives, controls, specialist workspace separation, and inherited workspace defaults
- Support AI-assisted delivery through narrow specialist roles rather than one generic agent
- Make future projects cloneable from a single proven master template
- Ensure derived projects inherit Forge Pipeline sync, self-improvement, self-reflection, and Quarto-aware documentation defaults unless explicitly overridden

## 5. Success Metrics

- Clone-to-first-governed-commit in under 30 minutes
- At least one real project launched from the template after MVP
- Zero critical governance findings in MVP project review
- 100% of governed deliverables linked to pipeline state and risk history
- Product Readiness Pack accepted by enablement reviewers as clear and useful

## 6. Scope

### In scope for MVP

- Master OpenClaw planning workspace
- Future GitHub template repo definition
- forge-* workspace model
- lifecycle from strategy to closure
- dedicated ForgeComms, ForgeTraining, and ForgeRisk domains
- document templates and schema planning
- governance gates and seeded risk model
- MVP backbone roadmap

### Out of scope for MVP

- full multi-repo orchestration
- full field-level data catalog automation
- automated ISO 27001 or NIST attestation
- advanced visual asset generation
- deep real-time control-plane integration beyond planned stubs

## 10. Operating Model

Project Foundry is based on a governed AI operating model.

It uses specialist AI roles for planning, drafting, orchestration, review, communications, training, privacy analysis, and risk analysis, while retaining a smaller human governance layer for approval, accountability, and exception handling. Privacy remains a dedicated specialist function, not folded invisibly into security.

## 11. Current Status

This project is still in planning.

No implementation should begin until planning gates, charter, roles, and initial risk visibility are in place.

## 12. Target Outcome

A reusable master workspace and future GitHub template repository that can be cloned to create new OpenClaw project environments with built-in lifecycle structure, governance, communications, training, risk, and delivery discipline.
