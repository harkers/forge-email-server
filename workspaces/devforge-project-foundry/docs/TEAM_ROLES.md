# DevForge: Project Foundry
## Human Governance Roles and AI Operating Roles

Status: Planning  
Date: 2026-03-28

## Intent

Project Foundry is designed around a governed AI operating model.

That means two layers must be kept distinct:
- a small **human governance layer** for approval, authority, and final accountability
- a larger **AI operating layer** made up of specialist roles that perform planning, drafting, orchestration, review, risk analysis, communications, training, and related delivery work

## Human Governance Layer

These are not intended to be a giant human staffing chart. They are the minimum governance roles needed to supervise and approve the operating model.

| Governance Role | Purpose |
|---|---|
| Executive Sponsor / Human Authority | Final authority for concept approval, funding, go / no-go decisions, and strategic escalation |
| Product Decision Owner | Final human decision-maker for scope, direction, and trade-offs |
| Approval Owners | Human sign-off points for governance, privacy, security, release, or exception decisions where required |

## AI Operating Layer

These are the primary specialist roles in the Project Foundry operating model.

| AI Operating Role | Purpose |
|---|---|
| Strategy Analyst | strategy shaping, problem framing, success measures |
| Architecture Analyst | structure, boundaries, design coherence |
| Governance / Documentation Curator | governance alignment, control shape, documentation coherence |
| Security Analyst | security expectations, trust boundaries, secure handling review |
| Privacy Analyst Agent | privacy-by-design input, privacy review support, DLP concerns |
| Risk Analyst | risk identification, evaluation, mitigation visibility |
| Orchestration Specialist | orchestration and structural setup support |
| Pipeline Controller | workflow state, sequencing, routing support |
| Comms Distiller | distillation, stakeholder-facing summaries, handoff packaging |
| Training Analyst | enablement structure, training packs, readiness outputs |
| Review Assistant | correctness review support and review packaging |
| Release Automation Agent | release-readiness and release-pack support |

## Privacy Position

Privacy remains a dedicated specialist role in the AI operating layer and should also have an explicit human approval path where privacy risk or sensitivity justifies it.

## Principle

Do not confuse AI operating roles with human staffing. Project Foundry uses specialist agent roles for execution and analysis, but retains human governance for approval, accountability, and exception handling.
