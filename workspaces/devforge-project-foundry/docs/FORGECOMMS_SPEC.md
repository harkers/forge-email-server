# DevForge: Project Foundry
## ForgeComms Specification

Status: Planning  
Date: 2026-03-28

## What ForgeComms Is

ForgeComms is the dedicated communications, distillation, and handoff domain.

It is not a generic messaging helper. It is the stage that takes raw outputs from major lifecycle phases and converts them into audience-ready, review-ready, and handoff-ready written artifacts.

## Purpose

- distil phase outputs
- apply style rules
- package handoffs
- produce stakeholder-ready summaries
- improve clarity without inventing status

## Core Outputs

- strategic summary
- discovery summary
- architecture summary
- governance outcome note
- sprint summary
- review outcome note
- release summary
- closure summary
- remediation brief
- stakeholder update
- handoff pack

## Controls

- COM-01 Communications and Stakeholder Control
- COM-02 Distillation and Handoff Communication Control
- AUD-01 Audit Trail and Traceability

## Key Principle

Raw outputs should not be passed directly to stakeholders or next-stage owners when distillation is required. ForgeComms exists to turn system exhaust into useful communication.
