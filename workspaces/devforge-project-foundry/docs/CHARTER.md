# DevForge: Project Foundry
## Project Charter

Status: Planning  
Date: 2026-03-28

## Purpose

Create the master OpenClaw workspace and future GitHub template repository used to generate governed project foundations with built-in lifecycle structure, controls, communication, training, risk, review, and release discipline.

## Problem

Projects often start with empty repositories, ad-hoc delivery habits, weak governance, fragmented documentation, and poor handoffs. That creates avoidable rework, inconsistent quality, slow onboarding, and weak traceability.

## Goal

Provide a reusable delivery foundation that helps new projects begin with strong structure rather than rebuilding process, folders, templates, controls, and role models from scratch each time.

## Objectives

- standardise the structure of OpenClaw-led projects
- define a repeatable lifecycle from strategy to closure
- create dedicated forge-* execution domains
- build in comms, training, and risk as first-class concerns
- support specialist AI-assisted roles with clear boundaries
- make the future repository cloneable and reusable

## Scope

### In
- planning and definition of the master workspace
- lifecycle, controls, roles, and workspace model
- future template repo design
- MVP roadmap and approval gates
- risk seed and governance position

### Out for now
- implementation of full automation
- live GitHub repo bootstrap before approval gates
- advanced policy engine work
- full ISO/NIST mapping implementation
- multi-project production rollout before pilot

## Constraints

- planning-first approach
- repository bootstrap only after approval gates
- privacy must have explicit ownership
- internal execution domains should use consistent forge-* naming
- MVP should prove the governed loop before deeper automation

## Success Measures

- clear planning pack approved by stakeholders
- future repo can be created from agreed structure without rethinking fundamentals
- first pilot project can be created from the template with minimal structural churn
- comms, training, and risk are present from the start rather than bolted on later

## Approval Intent

This charter exists to align stakeholders before implementation begins.
