# ForgeDiscord Visual Design Guide

## Brand Essence

ForgeDiscord is the communication and notification layer for the Forge ecosystem.
It should feel like a live operator surface: structured, calm, premium, and always readable.

If ForgeOrchestra is command, ForgeDiscord is signal.

## Core Attributes
- Intelligent
- Controlled
- Signal-first
- Operationally legible
- Quietly premium
- Systemic rather than social-noisy

## Should not feel
- Gamer-server clutter
- Neon Discord theme chaos
- Cheap bot dashboard UI
- Generic moderation panel design
- Alert-fatigue visual design

## Visual Positioning
- Apple: restraint and finish
- OpenAI: calm intelligence
- Linear: operational precision
- Modern observability tools: signal hierarchy

## Colour System
### Base
- Obsidian Black: `#07090D`
- Carbon Ink: `#0B1020`
- Deep Graphite: `#111827`
- Slate Glass: `#161D2E`

### Text
- Primary: `#F5F7FB`
- Secondary: `rgba(245,247,251,0.72)`
- Tertiary: `rgba(245,247,251,0.48)`

### Accents
- Signal Cyan: `#6EDFF6`
- Platinum Blue: `#7AA6FF`
- Neural Violet: `#7C72FF`
- Alert Amber: `#F6B36E`

## Layout
- Spacious shell
- High signal contrast
- Clean status grouping
- Timestamp-friendly components
- Thread/event/message cards should prioritise state and actionability

## UX Rules
- Show what changed, when, and whether it matters
- Prefer grouped alerts over floods
- Summaries must be scannable in seconds
- Status colours should be restrained and semantic

## Component Ideas
- Channel health cards
- Alert queue
- Notification digest panel
- Escalation timeline
- Thread status board
- Command palette
- Operator summary strip
