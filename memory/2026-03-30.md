### Project Summary

A lightweight internal platform for Google Compute Engine VM governance and operations:
- **Execution Layer**: Compute Engine API (`compute.googleapis.com`)
- **Workflow Layer**: Gmail API (approvals, notifications)
- **Evidence Layer**: Drive API (runbooks, evidence packs)
- **Register Layer**: Sheets API (VM inventory, change log)
- **Awareness Layer**: Service Health API (Google Cloud disruption events)

### Solo Operating Model

1. No destructive action without confirmation
2. No create action without register entry
3. No "success" state until operation resource confirms completion
4. No important action without evidence written to Drive
5. No silent failures; every failure generates a visible alert

### MVP Scope (6 things)
1. VM register (Sheets)
2. VM actions (create/start/stop/restart/inspect)
3. Action logging
4. Evidence records (Drive)
5. Gmail confirmations/notifications
6. Service Health context (Phase 2)

### Phase Plan
- **Phase 1**: Design & foundation — freeze MVP, enable APIs, define auth, register schema, evidence structure
- **Phase 2**: Thin end-to-end flow — request → confirm → execute VM → poll operation → update Sheets → write Drive evidence → send Gmail result
- **Phase 3**: Broaden action set — add remaining VM actions, build operator dashboard
- **Phase 4**: Operational hardening — retry/backoff, quota-aware scheduling, weekly reports
