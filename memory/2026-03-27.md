| Type | Status | Result |
|------|--------|--------|

| `q4_1` | ❌ Fails | No output/crashes |
| `turbo3` | ❌ Crashes | `GGML_ASSERT(obj_new) failed` in ggml.c:1760 |
| `turbo4` | ❌ Crashes | Same assertion failure as turbo3 |

### Error Details
Both turbo3 and turbo4 fail during KV cache initialization:
```
/opt/llama-cpp-turboquant/ggml/src/ggml.c:1760: GGML_ASSERT(obj_new) failed
```

This suggests the turboquant KV cache quantization types are not properly implemented in this fork, or there's a memory allocation bug when creating the KV cache with these types.

### Conclusion
The "turboquant" feature (custom KV cache quantization) is **non-functional** in this build. Only standard types (f16, f32, bf16) work properly. The build still provides CUDA acceleration and standard GGUF inference, but the specialized quantization doesn't work.
