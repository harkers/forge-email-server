# UK GDPR DSAR Deadline Tracking

## Core timing rule

- Standard deadline: **one calendar month** from receipt of the request.
- If the corresponding date does not exist in the following month, use the last calendar day of that month.
- If the request is complex or numerous, the deadline may be extended by **up to two further months**.
- If extending, notify the requester within the original one-month period and explain why.

## Operational fields to track per case

- Date received
- Date identity clarification requested
- Date identity clarification completed
- Date scope clarification requested
- Date scope clarification received
- Standard deadline
- Extension invoked? (yes/no)
- Extension notice date
- Extended deadline
- Final response date

## Suggested handling notes

- Record whether the request is valid on receipt or whether identity clarification is needed.
- Record the basis for any extension.
- Record any pauses or uncertainties as notes, not assumptions.
- Escalate edge cases for legal review.

## Simple examples

- Received **24 March 2026** → standard deadline **24 April 2026**
- Received **31 January 2026** → standard deadline **28 February 2026**
- Received **30 November 2026**, extended lawfully → up to **30 January 2027**
