# Standard Case Layout

```text
requests/<case-id>/
├── attachments/
├── identity-check.md
├── request.md
├── response.md
├── review-notes.md
├── scope.md
└── search-log.md
```

## Suggested case naming

Use a predictable format such as:

- `2026-001-jane-doe`
- `2026-002-employee-a`
- `2026-003-customer-export`

## File intent

- `request.md`: intake facts and deadlines
- `identity-check.md`: verification steps and outcome
- `scope.md`: defined scope, inclusions, exclusions, clarifications
- `search-log.md`: system-by-system audit trail
- `review-notes.md`: legal/privacy review notes and redaction decisions
- `response.md`: final response draft or sent copy
```
