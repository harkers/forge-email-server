# Integration — Self-Reflection + Self-Improving

## Recommended Relationship

Use `self-reflection` as the evaluator and `self-improving` as the memory layer.

```text
meaningful task
→ self-reflection runs
→ reusable lesson extracted
→ lesson routed by scope
→ self-improving stores and tracks it
→ promotion happens only after repetition or confirmation
```

## Why Split the Roles

This separation keeps the system clean:
- reflection stays lightweight and judgment-focused
- self-improving stays responsible for persistence, promotion, and decay

Without the split, every reflection risks becoming permanent memory too early.

## Suggested Self-Improving Handoff Format

Use a concise handoff such as:

```text
TYPE: reflection-candidate
SCOPE: domain/privacy
LESSON: Start incident assessments with a short conclusion block before the detailed analysis.
SOURCE: post-task review after privacy incident assessment
CONFIDENCE: medium
```

## Operating Principle

Reflect often enough to improve quality.
Store rarely enough to keep memory clean.
